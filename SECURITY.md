# Security Policy

## Supported Versions

Security fixes are provided for the latest released version of Keysweep. Older versions may not receive updates.

## Reporting a Vulnerability

If you discover a security vulnerability in Keysweep, **please do not open a public GitHub issue.**

Instead, report it privately by the following method:

- Open a private security advisory through GitHub.

Please include:

- A description of the vulnerability.
- Steps to reproduce the issue.
- The affected version.
- Any proof-of-concept or supporting information.

## Response Process

After receiving a report, we will:

1. Acknowledge receipt of the report.
2. Investigate and validate the issue.
3. Develop and test a fix if necessary.
4. Release the fix and publicly disclose the vulnerability once users have had an opportunity to update.

## Scope

This policy applies to vulnerabilities in the Keysweep source code.

Issues related to third-party dependencies should also be reported to the maintainers of those respective projects when appropriate.

## Responsible Disclosure

Please avoid publicly disclosing vulnerabilities until a fix has been made available. Responsible disclosure helps protect users while the issue is being addressed.
